Installing - top

ZX Point of Sale v1.3 is designed to work with Zen Cart 1.5.7a (latest version at the time of release). It PROBABLY can be used with older versions of Zen Cart, but installation is done at your own risk and no support will be provided in such cases.

Before you proceed, make a full backup of your site's files AND database. Installation is done at your own risk.

Unzip the files from "files" directory to your computer. Connect to your server via FTP. Check your admin directory name and rename the one in this package accordingly. Find and rename all instances of YOUR_TEMPLATE to the actual directory name of the template you're using. Now, upload your files using your FTP client (I recommend using Filezilla) to the store root. Be careful, the store root directory on the server is where your store is installed, not necessarily domain root.

There are several files that get overwritten and are considered as core files:

  • includes/classes/shopping_cart.php
  • includes/modules/pages/checkout_confirmation/header_php.php
  • includes/modules/pages/checkout_success/header_php.php
  • includes/modules/pages/login/header_php.php
  • includes/modules/pages/logoff/header_php.php
  • includes/modules/pages/shopping_cart/header_php.php
  • YOUR_ADMIN/includes/header.php
  • YOUR_ADMIN/includes/modules/product/collect_info.php
  • YOUR_ADMIN/includes/modules/update_product.php

In case you have made any changes to this file, make sure you merge it carefully (you can use a tool like WinMerge). All changes have been commented. Otherwise, you can simply overwrite.

After all files are in place, log in to your admin section and go to Tools->Install SQL Patches. Open the sql.txt that comes with this plugin, copy the contents, paste it in the admin section and click the "send" button.

Go to admin->Modules->Shipping, locate "In-store Pickup", click on it and click the "install" button on the right. This is a dedicated shipping module for this plugin and it will only show when going through checkout using the POS. It will not show for standard online customers.

Go to admin->Modules->Payment, locate "In Store Payment", click on it and click the "install" button on the right. This is a dedicated payment module for this plugin and it will only show when going through checkout using the POS. It will not show for standard online customers.

Go to the store frontend and create a "dummy" account for purchases that will not be linked to any customer account. You can call it "instore sales", set email to whatever you like (preferebaly something on your domain) and set it to the same state as your store is located in (for tax related calculation). This account will be used for all purchases where you select "Guest" and don't need customer's details to checkout.

Go to admin->Configuration->ZX Point of Sale and enter the Customer account password. This is a Master Password that MUST be used by any of your admin accounts. You can create a separate admin profile for these purposes or just use your own password. Next, enter the Guest account email address which is actually the email address used to create the "dummy" account.
Remember to update this entry every time you change the admin's password. If you create a separate admin profile, keep in mind the password will expire in 90 days.
IMPORTANT: you must go to admin->Configuration->My Store and set either "Customer Place Order: Single Admin ID" or "Customer Place Order: Admin Profiles" to enable using Master Password.

Stock by Attributes plugin support

Click here IF your site uses the Stock by Attributes plugin (SBA)

If you're using the Stock by Attributes plugin (SBA), this plugin supports it, but requires a few modifications to add a barcode field for SBA items. SBA comes with "Custom ID" field which is used as model number by ZX POS. If you want to use barcodes, some of the SBA files need to be modified.

Open the "SBA support" folder that comes with this plugin. You'll see there is a SBA zip file included - this is reference only because the modifications were based on THAT version of SBA. However, the SBA plugin is updated quite frequently so your files might not be the same - thus the reference file.

If you didn't modify the SBA plugin and if you're sure your site is using the exact same version of SBA, you can simply upload the contents of "files" directory to your store's root (remember to rename YOUR_ADMIN directory).
In case you're not sure about your SBA version, simply compare the files with the ones you have on your server and merge the changes. All changes are commented.

After all files are in place, log in to your admin section and go to Tools->Install SQL Patches. Open the sql-SBA.txt that comes with this plugin, copy the contents, paste it in the admin section and click the "send" button.

Usage - top

You need to log in to the admin section and click on the icon in the header section. This is the ONLY way to access POS features. The POS page is protected and can not be accessed without first logging into the admin section.

When adding products to your catalog, you can use your barcode scanner to enter the barcode.

You can add discounts directly in the POS interface. You can only enter numeric values, percentages will not be calculated.

The plugin comes with one dedicated shipping module and one dedicated payment module (In-store Pickup and In Store Payment). Both will be automatically selected during checkout, but you can select any of the other modules you have installed and use it for checkout.

You can print an invoice from the final checkout page (checkout_success). The plugin comes with a generic template which you can easily modify to suit your needs. There are two files you would need to edit:

  • store_root/posinvoice.php
  • store_root/posinvoice.css

Common Issues - top

I tested the plugin many times and it was also checked by other people, so it should work when you are using a well coded template and plugins. The only issues encountered so far were jQuery conflicts caused by badly coded templates. If that occurs, please make sure your jQuery scripts are loading properly and in a proper order.

If your site uses CSS/JS Loader, you will have to build the auto-loaders to avoid conflicts or broken jQuery scripts.

If your site uses any of the available checkout plugins (OPC, FEC, FEAC...), I suggest deleting auto_loaders/config.pos.shipping_observer.php file. By doing this, the POS In Store Pickup and In Store Payment modules will not be pre-selected.
